# Phase 03.5 — PLAN

**Goal:** Construir y distribuir una extensión Chrome MV3 que reemplaza el
paste en `web.whatsapp.com` por typing humano, controlada por marker
`__SCM_TYPE__:` desde el clipboard, con hotkey `Ctrl+Espacio`.

**Inputs:**
- `03.5-CONTEXT.md` — decisiones (hotkey, marker, naturalismo, distribución)
- `03.5-RESEARCH.md` — método primario `execCommand('insertText')` per char,
  con fallback `InputEvent` y técnica de envío via click del botón send

**Out of scope (deferred):** Ver `03.5-CONTEXT.md` → `<deferred>`. NO agregar
slider de intensidad, NI auto-update, NI soporte multi-sitio, NI métricas
de uso.

---

```

└── extensions/
    └── scm-paste-as-human/
        ├── manifest.json

        ├── lib/
```

Decisión de path: `GoogleSrapper/extensions/<nombre>/` como subdir, NO repo
separado. Razón: simplicidad para distribuir junto al panel (admin sirve
el zip desde `/api/admin/extension-download` en C-PANEL-01 si se quiere
después; fuera de scope por ahora).

### Task 1.2 — Módulo clipboard + marker parser
**File:** `lib/clipboard.js`

**Acceptance:**
- Función `async readMarkedClipboard()`:
  - Lee clipboard via `navigator.clipboard.readText()`
  - Si NO empieza con `__SCM_TYPE__:` → devuelve `{ valid: false }`
  - Si empieza con marker → devuelve `{ valid: true, text: <sin marker> }`
- Maneja error de permission (clipboard puede tirar `NotAllowedError` si
  el documento no tiene focus) → propaga error con mensaje legible
- Constante exportada `MARKER = "__SCM_TYPE__:"`

**Dependencies:** ninguna

---

### Task 1.3 — Motor de typing humano
**File:** `lib/typing.js`

**Acceptance:**
- Función `async humanType(text, { onProgress, signal })`:
  - Resuelve el editable: `document.querySelector('div[contenteditable="true"][role="textbox"]')`. Si no aparece, throw `'compose-not-found'`.
  - Focus + Selection collapsed al final (per RESEARCH.md pseudo-código)
  - Loop por char:
    - Si `signal?.aborted` → break (cancel)
    - Intenta `document.execCommand('insertText', false, char)`
    - Si retorna `false` → fallback `InputEvent('beforeinput', { inputType:'insertText', data: char, ... })` + `InputEvent('input', ...)`
    - Llama `onProgress(i, text.length)` después de cada char
    - Espera `delay(char, prevChar, i)` (ver siguiente bullet)
  - Función helper `delay(char, prev, i)`:
    - Base: `50 + random(0..100)` ms
    - Si `char` es `.,;!?:` → suma `150 + random(0..200)` ms
    - Cada N chars (random N entre 25 y 60): pausa "pensar" `1000 + random(0..2000)` ms
    - Typo simulado con probabilidad 2%: tipea char vecino del QWERTY → espera `100 + random(0..150)` → backspace via `execCommand('delete')` o `InputEvent('beforeinput', { inputType:'deleteContentBackward' })` → tipea char correcto. SOLO en chars alfabéticos. Implementar como helper `maybeTypo(char)` que devuelve la secuencia a tipear.
- NO toca Selection entre chars (deja que insertText avance el caret).
- Detecta IME activo: si `editable.matches(':composing')` o si llega
  `compositionstart` → throw `'ime-active'`.

**Notas técnicas:** El primer `execCommand` debe correr dentro del callstack
del hotkey handler (sin `setTimeout` previo) para preservar user-gesture.
Después el loop con `await sleep` está ok porque Lexical no exige gesture
para `insertText` después del primer trigger (assumption A2 del research —
si falla, el path fallback ya está cableado).

**Dependencies:** ninguna (es un módulo puro)

## Wave 2 — UI + control flow

### Task 2.1 — Badge flotante de progreso
**File:** `lib/badge.js`

**Acceptance:**
- `showBadge()` → inserta `<div id="scm-paste-badge">` en `document.body`
  con CSS inline (no contamina WA con clases). Posición fixed
  bottom-right, padding 8px 12px, `background: rgba(15,17,21,0.92)`,
  texto blanco, font-size 13px, border-radius 6px, z-index 99999.
  Contenido: `<span class="scm-label">Tipeando...</span> <span class="scm-progress">0/0</span>`
- `updateBadge(current, total)` → actualiza el `<span class="scm-progress">`
  a `${current}/${total}` y aplica width a `<span class="scm-bar-fill">`
  (barra de progreso como pseudo-element o nested span)
- `hideBadge()` → remueve el nodo
- Reusable: si ya existe, reusa; no crea duplicados

**Dependencies:** Task 1.1 (manifest) — para la inyección via content script

### Task 2.2 — Toast "Falta marker SCM"
**File:** `lib/toast.js`

**Acceptance:**

  con animación fade-in/out. Estilo coherente con el badge.

**Dependencies:** ninguna

---

### Task 2.3 — Cancelación + auto-pausa
**File:** `lib/cancel.js`

**Acceptance:**
- Función `createCancelController({ editable })` devuelve `{ signal, dispose }`:
  - `signal` es un `AbortSignal` (usa `AbortController` interno)
  - Listener en `document.addEventListener('keydown', ...)`:
    - Si tecla = Escape → `controller.abort('user-cancelled')`
    - Si tecla NO es de modificación pura (Shift/Ctrl/Alt/Meta solo) Y
      el target es el editable → `controller.abort('user-typing')`. Es decir,
      cualquier intento del usuario de teclear manualmente aborta para no
      pisar lo que escribe.
  - `dispose()` remueve los listeners
- El controller se crea fresh por cada invocación de typing (no global)

**Dependencies:** ninguna

## Wave 3 — Orchestration

### Task 3.1 — Content script entrypoint
**File:** `content.js`

**Acceptance:**
- Carga (vía `<script>` injection o concatenado en build) los módulos de
  `lib/`. Si quedamos sin bundler, concatenamos los archivos en build
  time → un único `content.js` final. Por ahora usar IIFE wrappers en
  cada lib y un `content.js` que los referencia globalmente (`window.__scmPaste = {...}`).
- Listener global `keydown`:
  - Match `Ctrl+Espacio` (`ev.ctrlKey && ev.code === 'Space'`)
  - `ev.preventDefault()` + `ev.stopPropagation()`
  - Llama `onHotkey()` (definido abajo) — debe correr SYNC en este callstack
    para preservar user-gesture del primer char.
- `async function onHotkey()`:
  1. `const clip = await readMarkedClipboard()` → si `!clip.valid`, `showToast('Falta marker SCM')` y return
  2. `const editable = document.querySelector('div[contenteditable="true"][role="textbox"]')` → si null, `showToast('No hay chat abierto')` y return
  3. `editable.focus()` + setea Selection (extraer helper en `typing.js`)
  4. `showBadge()`
  5. `const ctl = createCancelController({ editable })`
  6. **Trigger sync del primer char antes del await:**
     `const firstResult = document.execCommand('insertText', false, clip.text[0])`
     Esto preserva user-gesture. Después arranca el loop async para los chars restantes.
  7. `try { await humanType(clip.text.slice(1), { onProgress: updateBadge, signal: ctl.signal, _firstAlreadyTyped: true }) }
     finally { hideBadge(); ctl.dispose() }`
  8. Si error `'compose-not-found'`/`'ime-active'`/`'user-cancelled'`/`'user-typing'`: log a console + (opcional) toast suave. NO molestar al setter con modal.

**Dependencies:** Tasks 1.1, 1.2, 1.3, 2.1, 2.2, 2.3

## Wave 5 — Validation

### Task 5.1 — Test runbook manual
**File:** `extensions/scm-paste-as-human/TESTING.md`

**Acceptance:** documento con checklist de tests manuales:

1. **Install:** drag-drop del zip a `chrome://extensions/` (modo dev). Sin errores.
2. **Marker valid:** copiar desde panel SCM con botón "Copiar 👤" → ir a
   `web.whatsapp.com`, abrir un chat, focus en compose, Ctrl+Espacio →
   typing arranca, badge aparece, mensaje queda en el input.
3. **Send works:** después del typing, Enter (o click send button) → el
   mensaje se manda (aparece bubble con ✓✓). **CRÍTICO** — si esto falla,
   significa que Lexical no registró el state, fallback no funcionó →
   investigar.
4. **Marker missing:** copiar texto cualquiera (sin marker), Ctrl+Espacio
   → toast "Falta marker SCM", nada se tipea.
5. **Cancel Esc:** disparar typing largo, Esc a mitad → typing se detiene,
   badge desaparece, lo tipeado hasta ahí queda.
6. **Cancel manual:** disparar typing, empezar a teclear el setter →
   typing se detiene sin pisar lo del setter.
7. **Naturalismo visible:** observar que hay pausas en `.`, `,`, `?`,
   `!` y pausas más largas cada 25-60 chars. Confirmar que algún typo
   simulado aparece (ver console log si hace falta para verificar).
8. **No interfiere con Ctrl+V:** copiar y Ctrl+V normal → paste instantáneo
   funciona como siempre.
9. **No interfiere fuera de WA Web:** verificar que en otras pestañas
   Ctrl+Espacio no hace nada.
10. **Cap stress:** mensaje de 500 caracteres → completar sin crash, badge
    actualiza progreso.

**Pass criteria:** los 10 ítems en verde con al menos 2 setters reales
mandando 5-10 mensajes cada uno por al menos 1 día (24h) sin que ninguna
cuenta caiga en BANNED_TEMP atribuible a esta extensión.

**Dependencies:** todas las anteriores

| Risk | Mitigation | Owner |
|------|------------|-------|
| Lexical bloquea synthetic input → mensaje no se manda | Task 5.2 smoke test antes de todo. Fallback InputEvent ya planeado. | tech |
